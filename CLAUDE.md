# CLAUDE.md - 「思维笔记」博客设计规范

## 项目愿景

构建一个融合极简美学与创新交互的个人博客系统。这不仅是一个内容展示平台，更是一个思想交流的数字空间。通过流动的布局、呼吸的界面和沉浸式的阅读体验，让每一次访问都成为一场视觉与思维的双重享受。

### 核心目标

1. **内容优先**：以最优雅的方式呈现文字、代码和多媒体内容
2. **极简美学**：去除冗余，保留本质，让设计服务于内容
3. **流动体验**：所有交互如水般自然流畅，创造舒适的阅读节奏
4. **技术创新**：运用前沿技术打造独特的视觉效果和交互模式
5. **个性表达**：展现博主的技术品味和审美追求

## 设计理念

### 核心原则

1. **空间化**：将平面布局转化为具有深度的空间体验
2. **呼吸感**：界面随内容和交互自然呼吸，富有生命力
3. **单一焦点**：每个时刻只专注于最重要的内容
4. **液态交互**：所有过渡如水流般自然流畅
5. **情境适应**：根据内容类型自动调整视觉呈现

## 技术栈

### 核心框架

- **Next.js 15 (App Router)** - 利用最新的 React Server Components 和流式渲染
- **Tailwind CSS v4** - 新一代原子化 CSS，支持容器查询和动态主题
- **TypeScript** - 严格类型系统，确保代码质量和开发体验
- **HSL 颜色系统** - 基于 CSS 变量的动态主题，支持无限主题扩展

### 动画和交互

- **Framer Motion** - 声明式动画库，处理复杂的页面过渡和手势交互
- **Lenis** - 轻量级平滑滚动，提供丝滑的滚动体验

### 3D 和视觉效果

- **Three.js + React Three Fiber** - 创建 3D 文章空间导航和视觉效果
  - 文章节点的 3D 分布
  - 粒子效果和动态背景
  - WebGL 性能优化

### 内容管理

- **MDX** - 在 Markdown 中使用 React 组件，创建交互式内容
  - 自定义代码块组件
  - 交互式图表和演示
  - 嵌入式组件

## 页面设计规范

### 1. 首页（Landing Page）

#### 设计理念

首页是用户访问博客时的第一印象，它起着至关重要的作用，需要在瞬间传达出博客的独特气质，吸引用户进一步探索。

#### 视觉呈现

- **全屏欢迎区域**：该区域占据整个屏幕，为用户营造出强烈的视觉冲击。背景展示动态 3D 粒子效果或者流动的渐变，3D 粒子效果能够营造出一种科技感和未来感，而流动的渐变则可以带来柔和、灵动的视觉体验，这两种效果都能极大地增强页面的吸引力。
- **博客标题**：以超大字体呈现，这种大字体的设计能够让标题在页面中格外突出，迅速抓住用户的注意力。同时，标题会随鼠标的移动产生微妙的视差效果，即当用户移动鼠标时，标题会有轻微的位移，从而营造出一种立体的层次感，增强页面的交互性和趣味性。
- **简短的个人介绍**：采用打字机效果逐字显示，就像有人在现场一个字一个字地输入介绍内容一样，这种效果增加了页面的动态感和趣味性，使个人介绍不再枯燥，更能吸引用户的关注。
- **向下滚动指示器**：带有脉冲动画，该动画会有规律地闪烁或缩放，以此提示用户可以向下滚动页面继续浏览更多内容。

#### 交互设计

- **滚动背景粒子速度变化**：当用户滚动页面时，背景粒子的速度会相应地发生变化。在滚动过程中，粒子速度的改变能够营造出一种深度感，让用户感觉页面具有一定的空间层次，仿佛在一个立体的环境中浏览内容。
- **导航栏渐现效果**：导航栏在页面初始状态时是透明的，不会对页面的整体视觉效果造成干扰。当用户开始滚动页面后，导航栏会逐渐显现出来，这种设计既保证了首页整体视觉的简洁性，又能在用户需要导航时及时提供导航功能。
- **平滑过渡到文章列表区域**：从首页到文章列表区域的过渡采用平滑的效果，避免页面的突然跳转，给用户带来流畅、舒适的浏览体验。

### 2. 文章列表页（Articles）

#### 设计理念

摒弃传统的卡片网格展示方式，采用时间流的概念来展示文章。时间流的设计更符合人们对时间顺序的认知习惯，能够让用户更清晰地了解文章的发布时间顺序，同时也为页面带来一种独特的风格。

#### 视觉呈现

- **文章纵向排列**：文章以时间轴的形式纵向排列，每一篇文章都按照发布时间依次排列在页面上，就像一条时间线一样，用户可以很直观地看到文章的时间顺序。
- **文章占据视口比例**：每篇文章占据视口的 60 - 70%，这样的设计能够确保用户在浏览文章时，视线能够集中在当前文章上，减少其他元素的干扰，提高用户的专注度。
- **标题和副标题**：标题使用大字体，使其在页面中更加突出，能够让用户快速识别文章的主题。副标题包含文章的日期和阅读时长，这些信息能够帮助用户更好地了解文章的基本情况，从而决定是否要阅读该文章。
- **文章摘要渐变效果**：文章摘要以渐变透明度结束，即摘要的末尾部分透明度逐渐增加，这种效果暗示用户文章还有更多的内容等待他们去阅读，激发用户的阅读兴趣。
- **标签展示**：标签以极简的方式展示在文章的角落，既不影响文章的整体布局，又能让用户快速了解文章的相关标签信息。

#### 交互设计

- **当前文章标题固定**：当用户滚动页面时，当前文章的标题会固定在页面的顶部，形成一种自然的导航。这样，用户在滚动浏览文章内容时，始终能够看到文章的标题，方便他们随时了解当前阅读的文章。
- **文章区域扩张效果**：当鼠标悬停在文章区域时，文章区域会微微扩张，同时显示更多的预览内容。这种交互效果能够让用户在不进入文章详情页的情况下，了解更多关于文章的信息，增加用户与页面的互动性。
- **点击进入文章详情**：用户点击文章的任意位置，都可以进入文章的详情页，这种设计方便用户快速访问文章的详细内容。

### 3. 文章详情页（Article Detail）

#### 设计理念

致力于为读者提供极致的阅读体验，让读者能够完全沉浸在文章的内容中，减少外界因素的干扰。

#### 视觉呈现

- **Hero 形式展示标题**：文章标题以 Hero 形式展示，占据首屏的重要位置。Hero 形式通常具有较大的字体、醒目的颜色和合适的排版，能够让标题在页面中非常突出，吸引读者的注意力。
- **最优阅读宽度**：正文采用最优阅读宽度，即每行 65 - 75 字符。这种宽度能够保证读者在阅读时眼睛不需要大幅度地左右移动，减少阅读疲劳，提高阅读的舒适度。
- **段落间距动态调整**：段落间距会随内容的密度动态调整。对于内容较为密集的段落，段落间距会适当增大，以保证文字的可读性；而对于内容较为稀疏的段落，段落间距则会相应减小，使页面布局更加紧凑。
- **代码块功能**：代码块支持语法高亮和一键复制功能。语法高亮能够让代码更加清晰易读，不同的代码元素会以不同的颜色显示，方便读者区分和理解代码。一键复制功能则方便读者快速复制代码，提高效率。
- **图片查看功能**：图片支持全屏查看和说明文字。当用户点击图片时，图片会以全屏的形式展示，让用户能够更清晰地查看图片细节。同时，图片下方会配有说明文字，帮助用户更好地理解图片的内容。

#### 交互设计

- **阅读进度条**：阅读进度条极细，不会对阅读内容造成干扰。它能够实时显示用户的阅读进度，让用户了解自己已经阅读了多少内容以及还剩下多少内容未读。
- **目录导航高亮**：目录导航会自动高亮当前正在阅读的章节，方便用户随时了解自己在文章中的位置，同时也便于用户快速跳转到其他章节。
- **平滑滚动和惯性效果**：页面的滚动采用平滑滚动和惯性效果，即当用户滚动页面时，页面会平滑地移动，并且在滚动停止后会有一定的惯性继续滚动一小段距离，这种效果能够让用户的滚动操作更加自然、流畅。

### 4. 关于页面（About）

#### 设计理念

旨在展现个人特色，但在设计上保持克制和优雅，避免过于繁杂的设计影响页面的整体美感。

#### 视觉呈现

- **分屏设计**：采用分屏设计，左侧展示个人信息，右侧展示技能和经历时间线。这种左右分屏的设计能够将不同类型的信息清晰地分隔开来，方便用户查看。
- **CSS Grid 布局**：使用 CSS Grid 创建不对称但平衡的布局。CSS Grid 是一种强大的网格布局技术，通过它可以创建出复杂而又美观的布局。不对称的设计能够为页面带来独特的视觉效果，而平衡则保证了页面的整体稳定性和协调性。
- **技能图标展示**：技能以极简图标展示，这些图标简洁明了，能够快速传达技能的类型。当鼠标悬停在图标上时，会显示技能的详细信息，这种设计既保证了页面的简洁性，又能在用户需要时提供详细的内容。
- **联系方式按钮**：联系方式使用优雅的图标按钮，这些按钮的设计美观大方，与页面的整体风格相协调，能够吸引用户点击以获取联系方式。

#### 交互设计

- **滚动触发动画**：当用户滚动页面时，会触发渐进式动画。例如，一些元素会从隐藏状态逐渐显示出来，或者会有一些移动、缩放等动画效果，这些动画能够增加页面的动态感和趣味性。
- **时间线展开效果**：时间线会随滚动逐步展开，用户在滚动页面的过程中，能够逐步看到技能和经历的详细信息，这种设计增加了页面的交互性和探索性。
- **社交链接预览**：社交链接在鼠标悬停时会显示预览内容，例如链接的标题、简介等，让用户在点击链接之前能够了解更多关于链接的信息。

### 5. 项目展示页（Projects）

#### 设计理念

以视觉化的方式展示技术项目，通过直观的视觉效果让用户快速了解项目的特点和内容。

#### 视觉呈现

- **项目展示形式**：项目以大图配简介的形式展示，大图能够吸引用户的注意力，让用户快速对项目有一个直观的印象，而简介则能够提供项目的基本信息，帮助用户进一步了解项目。
- **CSS Grid 瀑布流布局**：使用 CSS Grid 创建瀑布流布局，瀑布流布局能够让项目在页面上自然地排列，形成一种错落有致的效果，既美观又能充分利用页面空间。
- **技术栈标签**：技术栈标签简洁明了，能够让用户快速了解项目所使用的技术，这些标签通常会以不同的颜色或样式区分不同的技术类型。
- **实时预览链接**：支持实时预览链接，用户点击链接即可在页面上实时预览项目的效果，无需跳转到其他页面，提高了用户了解项目的效率。

#### 交互设计

- **悬停显示详情**：当鼠标悬停在项目上时，会显示项目的详细信息，包括项目的功能、特点、技术实现等，让用户能够更深入地了解项目。
- **点击展开介绍**：用户点击项目时，会展开项目的详细介绍，详细介绍中会包含更多关于项目的信息，如项目背景、开发过程、成果等。
- **过滤和排序功能**：支持过滤和排序功能，用户可以根据自己的需求对项目进行过滤，例如按照技术栈、项目类型等进行过滤，也可以对项目进行排序，如按照项目的重要性、发布时间等进行排序。

### 6. 归档页面（Archive）

#### 设计理念

以最简洁的方式展示所有文章，突出时间的痕迹，让用户能够快速找到自己想要的文章。

#### 视觉呈现

- **按年份分组**：文章按年份分组，年份作为大标题显示在页面上，这种分组方式能够让用户快速了解文章的发布年份范围，方便用户根据年份查找文章。
- **文章信息显示**：每篇文章只显示日期和标题，这种简洁的显示方式能够让页面更加清爽，避免过多的信息干扰用户的查找。
- **等宽字体**：使用等宽字体营造档案感，等宽字体具有整齐、规范的特点，能够让文章列表看起来更加整齐有序，就像一份档案一样。
- **极简的悬停效果**：文章在鼠标悬停时会有极简的效果，例如颜色的轻微变化等，这种效果既不会过于突出影响页面的整体简洁性，又能让用户感受到交互的反馈。

#### 交互设计

- **快速跳转功能**：支持快速跳转到特定年份，用户可以通过点击年份标题或者使用年份选择器等方式快速跳转到自己想要查看的年份对应的文章列表。
- **搜索过滤功能**：提供搜索过滤功能，用户可以通过输入关键词来搜索特定的文章，也可以根据文章的标题、日期等条件进行过滤，提高查找文章的效率。

### 7. 搜索功能（Search）

#### 设计理念

追求即时、准确、优雅的搜索体验，让用户能够快速、方便地找到自己想要的内容。

#### 视觉呈现

- **全屏搜索界面**：采用全屏搜索界面，这种设计能够让用户将注意力完全集中在搜索输入框上，避免其他元素的干扰。
- **实时显示搜索结果**：在用户输入搜索关键词的过程中，会实时显示搜索结果，用户无需等待输入完成即可看到相关的搜索结果，提高搜索效率。
- **结果高亮匹配内容**：搜索结果中会高亮显示与搜索关键词匹配的内容，方便用户快速找到自己需要的信息。
- **分类显示**：搜索结果会分类显示，例如分为文章、标签、项目等类别，让用户能够更清晰地查看不同类型的搜索结果。

#### 交互设计

- **模糊搜索支持**：支持模糊搜索，即使用户输入的关键词不完全准确，也能够搜索到相关的内容，提高搜索的准确性和灵活性。
- **ESC 键退出**：用户按下 ESC 键即可退出搜索界面，方便快捷，避免用户在不需要搜索时还停留在搜索界面。

## 特色功能设计

### 1. 3D 文章宇宙（可选高级功能）

#### 设计理念

将文章关系可视化，创造一种探索式的阅读体验，让用户能够以一种全新的方式发现和阅读文章。

#### 实现要点

- **Three.js 创建 3D 空间**：使用 Three.js 库创建 3D 空间，Three.js 是一个用于创建和展示 3D 图形的 JavaScript 库，它能够方便地创建出复杂的 3D 场景。
- **文章节点分布**：文章作为发光节点分布在 3D 空间中，这些发光节点就像夜空中的星星一样，非常醒目，能够吸引用户的注意力。
- **相关文章光线连接**：相关的文章之间通过光线连接，这种光线连接能够直观地展示文章之间的关系，用户可以通过光线快速发现相关的文章。
- **鼠标拖拽旋转视角**：支持鼠标拖拽旋转视角，用户可以通过鼠标拖动来旋转 3D 空间的视角，从不同的角度查看文章节点，增加了探索的趣味性。
- **点击节点进入文章**：当用户点击某个文章节点时，会进入该文章的详情页，方便用户快速阅读文章。
- **性能优化**：采用 LOD（细节层次）和视锥剔除等性能优化技术。LOD 技术可以根据相机与节点的距离动态调整节点的细节程度，距离相机较近的节点显示更多的细节，而距离相机较远的节点则显示较少的细节，从而提高性能。视锥剔除技术则可以只渲染相机视野范围内的节点，减少不必要的渲染，提高渲染效率。

### 2. 主题系统

#### 预设主题

先支持亮色和暗色两种主题，亮色主题通常具有明亮、清新的视觉效果，适合在白天或光线充足的环境下使用；暗色主题则具有柔和、护眼的特点，适合在晚上或光线较暗的环境下使用。后续可以考虑增加更多自定义主题，满足不同用户的个性化需求。

### 3. 阅读增强

#### 专注模式

- **隐藏非必要元素**：在专注模式下，所有非必要的元素都会被隐藏起来，例如导航栏、侧边栏等，只保留阅读内容，让用户能够完全专注于文章的阅读。
- **自动调整字体大小和行距**：系统会根据用户的阅读习惯和设备屏幕大小自动调整字体大小和行距，以保证最佳的阅读体验。例如，在小屏幕设备上，字体大小会适当增大，行距也会相应调整，以提高文字的可读性。
- **背景轻微模糊**：背景会轻微模糊，这种效果能够突出当前段落，让用户的注意力更加集中在阅读内容上。
- **禅模式**：禅模式下，页面背景为完全空白，没有任何其他元素的干扰，为用户提供一种极致的专注阅读环境。

#### 进度追踪

- **细线进度条**：采用细线进度条实时显示阅读进度，细线进度条不会占用太多空间，同时又能让用户清晰地了解自己的阅读进度。
- **预计剩余阅读时间**：系统会根据文章的长度、阅读速度等因素预计剩余阅读时间，让用户对阅读所需的时间有一个大致的了解。
- **自动书签功能**：支持自动书签功能，当用户停止阅读时，系统会自动记录当前的阅读位置，下次打开文章时可以直接从该位置继续阅读。
- **阅读历史记录**：记录用户的阅读历史，用户可以随时查看自己阅读过的文章列表，方便回顾和继续阅读。

## 组件设计提示词

### 1. 通用组件设计原则

创建组件时，请遵循以下原则：

- 优先使用 shadcn ui 中的组件
- 使用 Framer Motion 实现流畅动画
- 支持暗色模式切换
- 响应式设计，移动优先
- 注重性能，避免不必要的重渲染
- 保持极简美学，去除冗余装饰

### 2. 核心组件提示词

**导航栏组件**：
"创建一个极简的导航栏组件，初始状态完全透明，滚动时渐现。包含 Logo、导航链接和主题切换按钮。使用 Framer Motion 实现平滑过渡，支持移动端汉堡菜单。"

**文章卡片组件**：
"设计一个呼吸感的文章卡片组件，默认显示标题、日期和摘要。鼠标悬停时轻微放大并显示阅读时长和标签。使用 CSS 变量实现主题适配，保持充足的留白。"

**阅读进度组件**：
"实现一个极细的阅读进度条，固定在页面顶部，宽度仅 2px。随页面滚动平滑填充，到达底部时触发完成动画。颜色使用主题强调色。"

**目录导航组件**：
"创建一个浮动的文章目录组件，自动提取标题层级，当前章节高亮显示。支持平滑滚动定位，在移动端可收缩为浮动按钮。"

**代码块组件**：
"设计一个增强的代码块组件，支持语法高亮、行号显示、一键复制。顶部显示语言类型和文件名（可选）。支持暗色主题切换。"

**搜索模态框组件**：
"实现一个全屏搜索界面，CMD+K 快捷键唤起。包含大尺寸输入框，实时搜索结果，支持键盘导航。使用模糊背景和平滑动画。"

**3D 文章节点组件**（高级）：
"使用 React Three Fiber 创建 3D 文章节点，节点发光效果，鼠标悬停时放大并显示标题。相关文章用光线连接，支持拖拽旋转视角。"

### 3. 页面布局提示词

**首页布局**：
"创建一个全屏欢迎区域，使用大标题和动态背景。向下滚动平滑过渡到文章列表。导航栏透明悬浮，滚动时渐现。整体采用大量留白，营造呼吸感。"

**文章页布局**：
"设计一个沉浸式阅读布局，内容区域最大宽度 720px 居中。使用黄金比例设置行高和段落间距。支持全屏图片查看，代码块横向滚动。侧边显示目录导航。"

**响应式断点**：
"基于内容而非设备设置断点：

- compact: < 640px (移动端)
- comfortable: 640px - 1024px (平板)
- spacious: > 1024px (桌面)
  使用 CSS 容器查询优化组件响应式。"

### 4. 动画和交互提示词

**页面过渡**：
"使用 Framer Motion 的 AnimatePresence 实现页面切换动画。内容从底部淡入，同时有轻微的上移效果。使用 layout 属性实现共享元素过渡。"

**滚动动画**：
"实现基于滚动的渐进式动画：元素进入视口时淡入，使用 Intersection Observer 提高性能。标题从左侧滑入，段落从底部淡入，保持动画时序。"

**微交互**：
"为所有可交互元素添加细腻反馈：

- 按钮：悬停时轻微上浮，点击时缩放
- 链接：下划线动画，从左到右展开
- 卡片：悬停时阴影加深，轻微放大
- 输入框：聚焦时边框发光"

### 5. 性能优化提示词

**图片优化**：
"使用 Next.js Image 组件，配置响应式尺寸，启用 lazy loading。对于 Hero 图片使用 priority 属性。支持 WebP 格式，提供降级方案。"

**代码分割**：
"对 3D 组件和重型库使用动态导入。非关键组件使用 React.lazy。确保首屏加载时间最短。"

**动画性能**：
"优先使用 transform 和 opacity 属性。避免触发重排的属性。使用 will-change 提示浏览器优化。在 prefers-reduced-motion 时禁用动画。"
