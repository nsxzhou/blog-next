# CLAUDE.md - 「思维笔记」博客设计规范

## 项目愿景

构建一个融合极简美学与创新交互的个人博客系统。这不仅是一个内容展示平台，更是一个思想交流的数字空间。通过流动的布局、呼吸的界面和沉浸式的阅读体验，让每一次访问都成为一场视觉与思维的双重享受。

### 核心目标

1. **内容优先**：以最优雅的方式呈现文字、代码和多媒体内容
2. **极简美学**：去除冗余，保留本质，让设计服务于内容
3. **流动体验**：所有交互如水般自然流畅，创造舒适的阅读节奏
4. **技术创新**：运用前沿技术打造独特的视觉效果和交互模式
5. **个性表达**：展现博主的技术品味和审美追求

## 设计理念

### 核心原则

1. **空间化**：将平面布局转化为具有深度的空间体验
2. **呼吸感**：界面随内容和交互自然呼吸，富有生命力
3. **单一焦点**：每个时刻只专注于最重要的内容
4. **液态交互**：所有过渡如水流般自然流畅
5. **情境适应**：根据内容类型自动调整视觉呈现

## 技术栈

### 核心框架

- **Next.js 15 (App Router)** - 利用最新的 React Server Components 和流式渲染
- **Tailwind CSS v4** - 新一代原子化 CSS，支持容器查询和动态主题
- **TypeScript** - 严格类型系统，确保代码质量和开发体验
- **HSL 颜色系统** - 基于 CSS 变量的动态主题，支持无限主题扩展

### 动画和交互

- **Framer Motion** - 声明式动画库，处理复杂的页面过渡和手势交互
- **Lenis** - 轻量级平滑滚动，提供丝滑的滚动体验

### 3D 和视觉效果

- **Three.js + React Three Fiber** - 创建 3D 文章空间导航和视觉效果
  - 文章节点的 3D 分布
  - 粒子效果和动态背景
  - WebGL 性能优化

### 内容管理

- **MDX** - 在 Markdown 中使用 React 组件，创建交互式内容
  - 自定义代码块组件
  - 交互式图表和演示
  - 嵌入式组件

## 页面设计规范

### 1. 首页（Landing Page）

**设计理念**：第一印象决定一切，首页应该立即传达博客的独特气质。

**视觉呈现**：

- 全屏欢迎区域，展示动态 3D 粒子背景或流动的渐变
- 博客标题以超大字体呈现，随鼠标移动产生微妙的视差效果
- 简短的个人介绍，使用打字机效果逐字显示
- 向下滚动指示器，脉冲动画提示用户继续浏览

**交互设计**：

- 滚动时背景粒子速度变化，营造深度感
- 导航栏初始透明，滚动后渐现
- 平滑过渡到文章列表区域

### 2. 文章列表页（Articles）

**设计理念**：摒弃传统的卡片网格，采用时间流的概念展示文章。

**视觉呈现**：

- 文章以时间轴形式纵向排列
- 每篇文章占据视口的 60-70%，确保专注
- 标题使用大字体，副标题包含日期和阅读时长
- 文章摘要以渐变透明度结束，暗示更多内容
- 标签以极简的方式展示在角落

**交互设计**：

- 滚动时当前文章标题固定在顶部，形成自然的导航
- 鼠标悬停时，文章区域微微扩张，显示更多预览内容
- 点击任意位置进入文章详情

### 3. 文章详情页（Article Detail）

**设计理念**：极致的阅读体验，让读者完全沉浸在内容中。

**视觉呈现**：

- 文章标题以 Hero 形式展示，占据首屏
- 正文采用最优阅读宽度（65-75 字符）
- 段落间距随内容密度动态调整
- 代码块支持语法高亮和一键复制
- 图片支持全屏查看和说明文字

**交互设计**：

- 阅读进度条极细且不干扰阅读
- 目录导航自动高亮当前章节
- 平滑滚动和惯性效果

### 4. 关于页面（About）

**设计理念**：展现个人特色，但保持克制和优雅。

**视觉呈现**：

- 分屏设计：左侧个人信息，右侧技能和经历时间线
- 使用 CSS Grid 创建不对称但平衡的布局
- 技能以极简图标展示，悬停显示详情
- 联系方式使用优雅的图标按钮

**交互设计**：

- 滚动时触发渐进式动画
- 时间线随滚动逐步展开
- 社交链接悬停时显示预览

### 5. 项目展示页（Projects）

**设计理念**：以视觉化方式展示技术项目。

**视觉呈现**：

- 项目以大图配简介的形式展示
- 使用 CSS Grid 创建瀑布流布局
- 技术栈标签简洁明了
- 支持实时预览链接

**交互设计**：

- 悬停时显示项目详情
- 点击展开详细介绍
- 支持过滤和排序功能

### 6. 归档页面（Archive）

**设计理念**：时间的痕迹，以最简洁的方式展示所有文章。

**视觉呈现**：

- 按年份分组，年份作为大标题
- 每篇文章只显示日期和标题
- 使用等宽字体营造档案感
- 极简的悬停效果

**交互设计**：

- 支持快速跳转到特定年份
- 搜索过滤功能

### 7. 搜索功能（Search）

**设计理念**：即时、准确、优雅。

**视觉呈现**：

- 全屏搜索界面，聚焦输入框
- 实时显示搜索结果
- 结果高亮匹配内容
- 分类显示（文章、标签、项目）

**交互设计**：

- 支持模糊搜索
- ESC 键退出

## 特色功能设计

### 1. 3D 文章宇宙（可选高级功能）

**设计理念**：将文章关系可视化，创造探索式阅读体验。

- 使用 Three.js 创建 3D 空间

**预设主题**：先支持亮色和暗色两种主题，后续可以考虑更多自定义主题

### 3. 阅读增强

**专注模式**：

- 隐藏所有非必要元素
- 自动调整字体大小和行距
- 背景轻微模糊，突出当前段落
- 禅模式：完全空白背景

**进度追踪**：

- 细线进度条
- 预计剩余阅读时间
- 自动书签功能
- 阅读历史记录

## 组件设计提示词

### 1. 通用组件设计原则

创建组件时，请遵循以下原则：

- 优先使用 shadcn ui 中的组件
- 使用 Framer Motion 实现流畅动画
- 支持暗色模式切换
- 响应式设计，移动优先
- 注重性能，避免不必要的重渲染
- 保持极简美学，去除冗余装饰

### 2. 核心组件提示词

**导航栏组件**：
"创建一个极简的导航栏组件，初始状态完全透明，滚动时渐现。包含 Logo、导航链接和主题切换按钮。使用 Framer Motion 实现平滑过渡，支持移动端汉堡菜单。"

**文章卡片组件**：
"设计一个呼吸感的文章卡片组件，默认显示标题、日期和摘要。鼠标悬停时轻微放大并显示阅读时长和标签。使用 CSS 变量实现主题适配，保持充足的留白。"

**阅读进度组件**：
"实现一个极细的阅读进度条，固定在页面顶部，宽度仅 2px。随页面滚动平滑填充，到达底部时触发完成动画。颜色使用主题强调色。"

**目录导航组件**：
"创建一个浮动的文章目录组件，自动提取标题层级，当前章节高亮显示。支持平滑滚动定位，在移动端可收缩为浮动按钮。"

**代码块组件**：
"设计一个增强的代码块组件，支持语法高亮、行号显示、一键复制。顶部显示语言类型和文件名（可选）。支持暗色主题切换。"

**搜索模态框组件**：
"实现一个全屏搜索界面，CMD+K 快捷键唤起。包含大尺寸输入框，实时搜索结果，支持键盘导航。使用模糊背景和平滑动画。"

**3D 文章节点组件**（高级）：
"使用 React Three Fiber 创建 3D 文章节点，节点发光效果，鼠标悬停时放大并显示标题。相关文章用光线连接，支持拖拽旋转视角。"

### 3. 页面布局提示词

**首页布局**：
"创建一个全屏欢迎区域，使用大标题和动态背景。向下滚动平滑过渡到文章列表。导航栏透明悬浮，滚动时渐现。整体采用大量留白，营造呼吸感。"

**文章页布局**：
"设计一个沉浸式阅读布局，内容区域最大宽度 720px 居中。使用黄金比例设置行高和段落间距。支持全屏图片查看，代码块横向滚动。侧边显示目录导航。"

**响应式断点**：
"基于内容而非设备设置断点：

- compact: < 640px (移动端)
- comfortable: 640px - 1024px (平板)
- spacious: > 1024px (桌面)
  使用 CSS 容器查询优化组件响应式。"

### 4. 动画和交互提示词

**页面过渡**：
"使用 Framer Motion 的 AnimatePresence 实现页面切换动画。内容从底部淡入，同时有轻微的上移效果。使用 layout 属性实现共享元素过渡。"

**滚动动画**：
"实现基于滚动的渐进式动画：元素进入视口时淡入，使用 Intersection Observer 提高性能。标题从左侧滑入，段落从底部淡入，保持动画时序。"

**微交互**：
"为所有可交互元素添加细腻反馈：

- 按钮：悬停时轻微上浮，点击时缩放
- 链接：下划线动画，从左到右展开
- 卡片：悬停时阴影加深，轻微放大
- 输入框：聚焦时边框发光"

### 5. 性能优化提示词

**图片优化**：
"使用 Next.js Image 组件，配置响应式尺寸，启用 lazy loading。对于 Hero 图片使用 priority 属性。支持 WebP 格式，提供降级方案。"

**代码分割**：
"对 3D 组件和重型库使用动态导入。非关键组件使用 React.lazy。确保首屏加载时间最短。"

**动画性能**：
"优先使用 transform 和 opacity 属性。避免触发重排的属性。使用 will-change 提示浏览器优化。在 prefers-reduced-motion 时禁用动画。"

## 设计系统文档

创建统一的设计系统，确保整个博客的一致性：

### 设计令牌（Design Tokens）

**颜色系统**：

- 使用 HSL 格式定义颜色
- 主色、背景色、文字色、边框色
- 每种颜色定义不同透明度变体

**字体系统**：

- 标题字体：优先系统字体，备选 Inter
- 正文字体：优先系统字体，中文优化
- 代码字体：JetBrains Mono 或 Fira Code

**间距系统**：

- 基于 8px 网格
- 定义统一的间距尺度
- 响应式间距调整

**动画系统**：

- 统一的缓动函数
- 标准动画时长
- 交互状态定义

### 组件库规范

所有组件应该：

1. 支持主题切换
2. 完全响应式
3. 可访问性优先
4. 性能优化
5. 类型安全
